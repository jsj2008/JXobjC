Superclass:
Cltn
Declared In:
OrdCltn.h

Introduction

An Ordered collection of objects.

Discussion

Stores objects in an ordered form similar to the C++ STL's Vector. Objects at any specified index may be retrieved, removed, or inserted. The index begins at 0 for the first object, and ends at size - 1 for the last.

add:

- add:anObject;
Parameters
anObject

Object to add.

Discussion

Add an object to the end of the collection.

addAll:

- addAll:aCltn;
Parameters
aCltn

Collection whose entries are to be added to the receiver.

Discussion

Adds each member of the specified collection to the receiver.

The specified collection may in fact not be a collection at all. It simply needs to respond to eachElement as a collection does. If the specified collection is nil, then no action is taken.

addFirst:

- addFirst:newObject;
Parameters
newObject

Object to add.

Discussion

Add an object as the first in the collection.

Any extant objects are accordingly moved up to the next higher offset.

addIfAbsent:

- addIfAbsent:anObject;
Parameters
anObject

Object to add.

Discussion

Add an object, but only if the same object isn't already there.

Equality here is determined by equal object addresses.

addIfAbsentMatching:

- addIfAbsentMatching:anObject;
Parameters
anObject

Object to add.

Discussion

Add an object, but only if a matching object isn't already there.

Matching is determined using the isEqual: method.

after:

- after:anObject;
Parameters
anObject

Object whose immediately-following neighbour is retrieved.

Discussion

Retrieves the object positioned immediately after anObject.

If the specified object is not found in the collection, a Could not find object exception is signaled. If the object is the last in the collection, nil is returned.

at:

- (id)at:(unsigned)anOffset;
Parameters
anOffset

Offset at which to retrieve an object.

Discussion

Retrieves the object positioned at the specified offset.

If the specified offset exceeds the size of the collection, an OutOfBounds exception is signaled. Offsets begin at 0 for the first object and end at the OrdCltn's size minus 1 for the last.

collect:

- collect:transformBlock;
Parameters
aBlock

Block with which to evaluate each object.

Discussion

Returns a new OrdCltn containing the result of applying a block to each object in the receiver in turn.

The block receives each object in turn as its sole parameter. The block should return another object. If the block returns nil, then for this object nothing is added to the new collection.

copy

- copy;
Discussion

Copies the OrdCltn. The objects themselves are not copied, only their identifiers, so the contents are 100% identical to the original OrdCltn.

includesAllOf:

- (BOOL)includesAllOf:aCltn;
Parameters
aCltn

Collection whose entries' presence are to be tested for in the receiving collection.

Return Value

YES if receiving collection includes every entry of aCltn, NO if not.

Discussion

Asks whether the collection includes all the items in another specified collection.

This is effected through sending an includes: message for each element. If every such message is answered YES, this message replies YES. If any are not found in the receiver, then the reply is NO.

includesAnyOf:

- (BOOL)includesAnyOf:aCltn;
Parameters
aCltn

Collection whose entries' presence are to be tested for in the receiving collection.

Return Value

YES if receiving collection includes any entry of aCltn, NO if not.

Discussion

Asks whether the collection includes any of the items in another specified collection.

This is effected through sending an includes: message for each element. If any of these messages are answered with YES, then the reply to this message is YES. If none, NO.

- (BOOL)isEqual:aCltn;
Discussion

Compares the OrdCltn with another collection.

If the specified object is another collection, iterates through each indexed position. If each element of both respond affirmatively to the isEqual: message in turn, the collections are considered equal.

remove:

- remove:oldObject;
Parameters
anObject

Object to be removed from the collection.

Discussion

Removes the exact object specified, returning it.

If the specified object could not be found, returns nil.

remove:ifAbsent:

- remove:oldObject ifAbsent:exceptionBlock;
Parameters
anObject

Object to be removed from the collection.

exceptionBlock

Block to be evaluated if anObject not found.

Discussion

Removes the exact object specified, returning it. If unfound, calls a block.

If the specified object could not be found, the specified exception block is called, the block's return value being returned. The remove: method can thus be defined as such: [someCltn remove: someObject ifAbsent:{ nil }];

removeAll:

- removeAll:aCltn;
Parameters
aCltn

Collection whose entries are to be removed from the receiver.

Discussion

Removes each member of the specified collection from the receiver.

Objects are removed if they are the same object (equal address) as an object in the specified collection. If the specified collection is nil, then no action is taken.
