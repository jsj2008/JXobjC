Introduction

Implements Blocks, which are often known as Lambda expressions or Enclosures. These couple a method with the environment in which it was defined. They can be used to similar effect to function pointers, but with a great deal more flexibility and simplicity.

You may implement a block accepting two id-type parameters and returning the answer to the self message sent to the second in this way: Block * aBlock = {:first :second | [second self]};

ifError:

- ifError:aHandler;
Parameters
aHandler

Block taking two arguments (:msg and :rcv) for handling the exception.

Return Value

The id-type return value of the block.

Discussion

Calls into the block. If an exception is raised, deploys a handler block with two arguments (the Message first, and then the object to which the sent message triggered an exception.)

on:do:

- on:aClassOfExceptions do:aHandler;
Parameters
aClassOfExceptions

The Exception class, or one of its subclasses.

aHandler

Block taking an argument (:except) for handling the exception.

Return Value

The id-type return value of the block.

Discussion

Calls into the block. If an exception is raised and matches the specified class of exceptions (the exception is an instance of the class, or of one of its subclasses,) a handler block is called with its single argument being the instance of the exception raised.
